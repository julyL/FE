#### web页面解析
```
1. css阻塞dom的解析,但会阻塞页面渲染  
// demo1中 block.js会先打印(说明没有阻塞解析),随后2秒后页面才显示。

2. js会阻塞dom的解析和渲染,但不会阻塞后续js的下载(会阻塞执行)      
// demo2 log.js很快就加载了但在5秒之后才打印,说明block.js?5000阻塞了dom解析

3. 下载css会阻塞外链js的执行(不会阻塞js的下载)    
// demo3  block.js很快就加载了,但是5秒之后才会执行

4. 添加defer的js不会阻塞页面的渲染
//  demo4 页面会先渲染,block.js?5000会后续再执行

注: 
dom解析之后才可以通过js获取到dom
dom渲染是将其显示在页面上的过程
```
